---
title: "BootstrapFinetune — LM Weight Fine-Tuning Optimizer"
sidebarTitle: "BootstrapFinetune"
description: "BootstrapFinetune generates a fine-tuning dataset from your program's traces and uses it to update model weights, enabling smaller models to match larger ones."
---

Prompt optimization improves a model's behavior by changing the instructions it receives, but it leaves the model's weights untouched. `BootstrapFinetune` goes one step further: it runs your program on the training set, collects the traces where the output passes your metric, converts those traces into a supervised fine-tuning dataset, and submits that dataset to your LM provider to update the model's weights. This is the right approach when you need to compress knowledge from a large teacher model into a smaller, cheaper model — or when the task is complex enough that no prompt alone reliably elicits the correct behavior.

## Import

```python
from dspy.teleprompt import BootstrapFinetune
```

---

## Constructor

```python
optimizer = BootstrapFinetune(
    metric=None,
    multitask=True,
    train_kwargs=None,
    adapter=None,
    exclude_demos=False,
    num_threads=None,
)
```

### Parameters

<ParamField path="metric" type="callable" default="None">
  A scoring function with signature `(example, prediction, trace) -> float | bool`. Only traces that pass this metric are included in the fine-tuning dataset. When `None`, all collected traces are used.
</ParamField>

<ParamField path="multitask" type="bool" default="True">
  When `True`, all predictors that share the same LM are fine-tuned on a single combined dataset. When `False`, each predictor is fine-tuned independently on its own data. Multitask mode is more data-efficient and usually produces better results.
</ParamField>

<ParamField path="train_kwargs" type="dict" default="None">
  Provider-specific keyword arguments forwarded to the LM's fine-tuning API. Use this to set hyperparameters such as learning rate or number of epochs that your provider exposes. Accepts either a flat dict (applied to all LMs) or a dict keyed by `dspy.LM` instance for per-model configuration.
</ParamField>

<ParamField path="adapter" type="Adapter" default="None">
  The data-formatting adapter used when serializing traces into fine-tuning examples. Defaults to `ChatAdapter`. Accepts either a single adapter instance (applied to all LMs) or a dict keyed by `dspy.LM` instance.
</ParamField>

<ParamField path="exclude_demos" type="bool" default="False">
  When `True`, few-shot demonstrations are stripped from the fine-tuning examples, producing a zero-shot fine-tuned model. When `False`, any demos present in the predictor at compile time are included in each training example.
</ParamField>

<ParamField path="num_threads" type="int" default="None">
  Number of parallel threads used when bootstrapping traces from the training set. Defaults to `dspy.settings.num_threads`.
</ParamField>

## Setting the LM Explicitly

Because `BootstrapFinetune` fine-tunes the specific LM attached to each predictor, you must set the LM on your student program directly via `set_lm()` rather than relying on the global `dspy.configure()` setting. If you skip this step, `BootstrapFinetune` will raise a `ValueError` at compile time.

```python
from dspy.teleprompt import BootstrapFinetune
import dspy

lm = dspy.LM('openai/gpt-4o-mini', api_key='...')
program = MyProgram()
program.set_lm(lm)  # required

optimizer = BootstrapFinetune(
    metric=my_metric,
    train_kwargs={'n_epochs': 2},
)

finetuned = optimizer.compile(program, trainset=train)
finetuned.save('program_finetuned.json')
```

---

## `compile()` Method

`compile(student, trainset, teacher=None)` bootstraps traces from the training set, builds the fine-tuning dataset, submits the fine-tuning job to your provider, waits for it to complete, and returns the original `student` program updated so that each predictor's LM now points to the fine-tuned model checkpoint.

```python
finetuned = optimizer.compile(student, trainset=train, teacher=None)
```

<ParamField path="student" type="dspy.Module" required>
  The program to fine-tune. All predictors inside the program must have their LM set via `set_lm()` before calling `compile()`.
</ParamField>

<ParamField path="trainset" type="list[dspy.Example]" required>
  Training examples used to generate the fine-tuning dataset. More examples generally produce better fine-tuned models; aim for at least a few hundred passing traces.
</ParamField>

<ParamField path="teacher" type="dspy.Module" default="None">
  An optional pre-compiled teacher program used to generate higher-quality bootstrapped traces. When `None`, the student itself is used as the teacher. Pass a stronger, already-optimized program here to distill its behavior into the student's weights.
</ParamField>

---

<Warning>
  Fine-tuning modifies model weights on your provider's infrastructure — charges apply. Check your provider's fine-tuning pricing before running `BootstrapFinetune`, and keep in mind that the cost scales with dataset size and the training hyperparameters you configure via `train_kwargs`.
</Warning>

<Note>
  Not all providers support weight fine-tuning. The OpenAI fine-tuning API and HuggingFace are the most common targets. Local Ollama models typically do not support fine-tuning — for local workflows, consider using prompt optimization with `MIPROv2` or `GEPA` instead.
</Note>

<Tip>
  Combining `BootstrapFinetune` with a prompt optimizer in a `BetterTogether` pipeline often yields the best results. Run prompt optimization first to identify the highest-quality traces, then fine-tune on those traces to bake the behavior into the model's weights.
</Tip>
